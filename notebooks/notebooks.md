## PyTorch note (NB05d)

PyTorch is **not included** in `requirements.txt` or `environment.yml` because its installation depends on your hardware. Install it separately after activating the environment:

```bash
# CPU-only (works everywhere, recommended for students):
pip install torch --index-url https://download.pytorch.org/whl/cpu

# GPU — CUDA 12.1 (requires NVIDIA GPU + driver ≥ 525):
pip install torch --index-url https://download.pytorch.org/whl/cu121
```

**Benchmark (NB05d scale — MLP, 2 000 rows, 100 epochs):**

| Device | Time |
|---|---|
| CPU | ~0.52 s |
| GPU (RTX 3090, CUDA 12.1) | ~0.27 s |

At NB05d's dataset scale (2 000 rows, 11 features), the GPU offers ~2× speedup. This gap grows significantly with larger datasets. For this course, CPU-only is perfectly adequate.

## Warnings (non-blocking)

### NB06 — cells 50 and 54: `FutureWarning` from SHAP

```
FutureWarning: The NumPy global RNG was seeded by calling `np.random.seed`.
In a future version this function will no longer use the global RNG.
Pass `rng` explicitly to opt-in to the new behaviour.
```

**Cause:** `shap.summary_plot()` internally calls `np.random.seed`. This is a SHAP library issue (v0.51) that will be fixed in a future SHAP release.  
**Impact:** None — the plot renders correctly.  
**Student action:** Ignore this warning. It does not affect results.

### NB06 — cell 50: `TqdmWarning`

```
TqdmWarning: IProgress not found. Please update jupyter and ipywidgets.
```

**Cause:** `ipywidgets` is not installed. The SHAP progress bar falls back to text mode.  
**Impact:** None — SHAP runs correctly without the widget.  
**Student action:** Ignore. Optionally install with `pip install ipywidgets`.

## Google Colab compatibility

| Risk | Severity | Resolution |
|---|---|---|
| Missing packages (`shap`, `eurostat`, `world_bank_data`) | Medium | `!pip install -q shap eurostat world_bank_data` at the top of the notebook |
| Data files not accessible | **High** | Upload CSV(s) via Files panel, or mount Google Drive (see below) |
| `DATA_DIR` path does not resolve | **High** | Override the `DATA_DIR` line (see below) |
| `torch` not installed | None | Colab ships PyTorch with GPU by default |
| `jupyter` / `ipykernel` not installed | None | Colab provides its own kernel |
| NB04 writes `dataset_C_attrition_model_ready.csv` | Low | Written to the Colab session — valid for the session, lost on disconnect |

### Quick Colab setup cell (add at the top of any notebook)

```python
# ── Colab setup ───────────────────────────────────────────────────────────────
import sys
if 'google.colab' in sys.modules:
    # 1. Install missing packages
    import subprocess
    subprocess.run(['pip', 'install', '-q', 'shap', 'eurostat', 'world_bank_data'])

    # 2. Mount Drive (recommended — avoids re-uploading every session)
    from google.colab import drive
    drive.mount('/content/drive')
    from pathlib import Path
    DATA_DIR = Path('/content/drive/MyDrive/ai-ml-finance/data/processed')

    # --- OR upload files manually and use: ---
    # DATA_DIR = Path('.')
```

> Replace `/content/drive/MyDrive/ai-ml-finance/data/processed` with the actual path
> where you saved the `data/processed/` folder in your Google Drive.
